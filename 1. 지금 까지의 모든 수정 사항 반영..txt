코드 파일에서 한 글자도 생략하면 절대 안됨.
전체 수정사항을 반영해서 코드를 새로 짜고 파일로 만들어야 함.
기존의 css js html 파일을 그대로 대체해도 전혀 이상 없게 정교하게 만들어야 함.
내가 지정하지 않은 내용이더라도 사용자가 이용하기에 불편하거나 이상한 모양이면 적절히 수정해야 함.
파일은 내가 다운 받을 수 있게 함. 
수정이 완료 될 때까지 나에게 따로 허락이나 확인 진행 여부를 물어보지 않음.
모든 조건이 완료된 뒤에 결과물을 업로드하고 마지막에 완료 됐다고 알려줘야함.
중간에 확인 절차 없음. 수정 방향 질문은 처음에 시작하기 전에 한 번에 마쳐야 함.
